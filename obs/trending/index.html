Pipeline Metrics

Events
Every frame received from the Pusher WebSocket since page load — includes pings, subscription confirmations, errors, AND chat messages. Climbing = WS is alive. Stuck at 0 = WS won't connect.
Chat
Subset of events that matched as chat messages (not pings or system frames). Climbing alongside Events = chat-event matcher is working.
Msgs
Chat messages we successfully parsed and counted into the sliding windows. Climbing alongside Chat = end-to-end pipeline working.
Users
Distinct chatters across ALL live streamer chats since page load. The people typing in chat — not the streamers, not the viewers. All-time count, not windowed.
If a number to the right of another stops climbing, that's where the pipeline breaks. Debug overlay auto-shows when something's wrong; force it with ?debug=1.
